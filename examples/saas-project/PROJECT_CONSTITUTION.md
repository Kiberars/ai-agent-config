## Problem Statement

Small teams lose track of priorities when juggling too many tasks across tools.
TaskFlow gives them a single inbox with AI-driven prioritization, so the most
important work always surfaces first.

## Architecture

### Structure
Monorepo. Next.js handles both frontend and API routes.

### Folder Layout
```
taskflow/
├── app/                  ← Next.js App Router pages
│   ├── (auth)/           ← Auth-gated routes
│   ├── api/              ← API route handlers
│   └── layout.tsx
├── components/           ← Shared UI components
├── lib/                  ← Business logic, utilities
│   ├── ai/               ← AI prioritization module
│   ├── db/               ← Prisma client + queries
│   └── email/            ← Email service
├── prisma/               ← Schema + migrations
├── tests/                ← Jest + React Testing Library
└── public/
```

### Key Design Decisions
- Server Components by default, Client Components only for interactivity
- All DB access via Prisma — no raw SQL outside migrations
- AI calls are async background jobs — never block the UI

### External Integrations
- Clerk (auth webhooks → user sync)
- OpenAI API (task prioritization)
- Stripe (subscription billing)
